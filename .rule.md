## 交互原则

- 有多种理解时，列出选项让我选，不要自行决定
- 存在更简方案时主动提出，可以反驳我的方案
- 不确定就停下来问，不要猜测后实现
- 只改需要改的，不要顺手"改进"周边代码或格式
- 不要添加未要求的功能、抽象层或防御性错误处理
- 发现不相关的死代码，提一嘴但不要删

## 变更验证

涉及 3 个以上文件或步骤的改动，先列计划：
```
1. [步骤] → 验证: [方法]
```
改完自行运行测试/lint 验证，不要等我确认。

## 语言

- 所有回答使用简体中文
- 技术术语保留英文，附中文解释
- 代码注释用中文
- 变量名、函数名、命令行、配置文件保持英文

## Python 规范

- 使用 Google 风格 docstring（Args/Returns/Raises 分段）
- 所有函数、类必须有中文 docstring
- 必须使用 type hints
- 单行不超过 120 字符

## Git 规范

- 打 tag 时自动更新 version 文件中的版本号
- commit message 格式: `类型: 描述`（如 `feat: 新增导出功能`）
- 类型: feat / fix / refactor / docs / chore
